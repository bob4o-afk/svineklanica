# Rubric Scorecard — Farm the Points Deliberately

> The judging criteria turned into a **live checklist**. Self-score at each gate (`08_master_plan.md`). The goal: don't *hope* you hit the rubric — *engineer* it. Weights are the real ones from the reglament.

## The 4 criteria + what each really tests

### 1. Technical Mastery & Security — **30%**
*"Does it actually work? Real data, not samples. Working demo? Code survives more than the presentation?"*
- [ ] Runs on **real Bulgarian data** (not mock/sample) — *the single most-repeated rule*
- [ ] **Live working demo** (not a video, not slides)
- [ ] Handles real-world mess (Cyrillic/UTF-8, missing values) without crashing
- [ ] Runs **locally / offline** (resilient to venue wifi)
- [ ] Code is real, not vaporware — repo is public + runnable
**Self-score /10:** ___  (≥7 = green)

### 2. Radical Critical Thinking — **30%**
*"Real problem or just a symptom? Potential to provoke real reaction from citizens, journalists, institutions?"*
- [ ] Bites a **systemic** problem, not a surface symptom
- [ ] Would make a **citizen/journalist actually react** (anger, share, act)
- [ ] Targets a **real, recurring** Bulgarian pain (not a once-off curiosity)
- [ ] The insight is **non-obvious** — not a "thousand-times-chewed" concept
- [ ] Every critique is **fact-backed** (data/source), no disinformation
**Self-score /10:** ___  (≥7 = green)

### 3. System Design & UX Revolution — **20%**
*"Solves a real pain of a real user? Can an ordinary, non-technical citizen use it?"*
- [ ] An **ordinary citizen** (non-technical) can use it unaided
- [ ] **Bulgarian** UI, clear labels, one obvious action
- [ ] Solves a pain the user **actually has**, recurring if possible
- [ ] You tested it on someone outside the team (a "mentor"/stranger)
**Self-score /10:** ___  (≥7 = green)

### 4. Presentation & "Roast" — **20%**
*"How sharp, funny, well-argued? Speaking with facts while provoking."*
- [ ] A wound-first **cold open** (number/contradiction)
- [ ] One unforgettable **punch** (the damning output)
- [ ] **Funny + sharp**, deadpan, fact-backed roast
- [ ] Every on-screen claim has a **source** ready for Q&A
- [ ] A **Bulgarian** punk reference woven in (*"Мутри вън"* etc.)
- [ ] Timed ≤ 9:00, rehearsed twice
**Self-score /10:** ___  (≥7 = green)

## How to spend effort (the math)
- **Tech + Radical = 60%.** A working demo on real data that bites a real problem is the bulk of your score. **Protect these first.**
- **UX + Roast = 40%, and they're cheap to win** relative to code: Bulgarian labels + a rehearsed punch line move the needle fast. Don't skip them for one more half-feature.
- **The trap:** pouring hour 30+ into a 5th feature (marginal Tech gain) instead of the pitch (full 20% still on the table). At each gate, fix your **lowest-scoring** criterion, not your favorite one.

## Disqualifiers (auto-zero — check you're clean)
- [ ] No unauthorized system access / illegal data
- [ ] No plagiarism — core built on-site
- [ ] Open-source license present (MIT)
- [ ] No disinformation — claims sourced
